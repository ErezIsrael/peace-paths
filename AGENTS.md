# ☮️ Peace Paths

AI-powered tracker of concrete peace initiatives across the Middle East.

[Live](https://peace-paths.pages.dev)

---

## ⚠️ Deployment Rules — READ FIRST

This site has **2 major components**:

1. **Pages (static frontend)** — stored and updated in the **GitHub repo**. Commit every change to `app/` (HTML, CSS, JS, `_headers`) to Git.
2. **Data files** — `solutions.json`, `categories.json`, etc. are **never committed to Git**. They are generated by the AI pipeline and uploaded directly to Cloudflare via `wrangler pages deploy`.

**NEVER use `wrangler` to upload files that live in the GitHub repo.** Commit them to Git instead. Only use `wrangler` for deploying data files.

---

## Architecture

```
29 RSS Feeds → ai-analyze-prod.py → llama.cpp (AI) → solutions.json → Cloudflare Pages
```

- **AI Pipeline** (`ai-analyze-prod.py`): Fetches 29 RSS feeds → single-article LLM inference → ME relevance filter → category classification → writes `solutions.json` → deploys via `wrangler pages deploy`.
- **Frontend** (`app/`): Static HTML/JS/CSS with solution cards, phase bars, momentum banner, and activity feed.
- **Admin Panel** (`admin/`): Manage categories in `categories.json` without editing files manually.

## Key Details

| Item | Value |
|------|-------|
| RSS feeds | 29 sources — defined in `rss-feeds.json` (gitignored) |
| LLM | llama.cpp — configured via `LLAMA_CPP_URL` env var |
| Categories | Defined in `categories.json` (gitignored) |
| Output | `app/solutions.json` — deployed via `npx wrangler pages deploy` |
| Env vars | `LLAMA_CPP_URL`, `AI_MODEL`, `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID` |

## Running the AI Pipeline

```bash
# Daily — full 7-day window, overwrite solutions.json
python ai-analyze-prod.py --daily

# Fast — last 2h, merge into existing data
python ai-analyze-prod.py --fast

# Skip upload (write local file only)
python ai-analyze-prod.py --fast --skip-upload

# Keyword fallback (skip AI, for testing)
python ai-analyze-prod.py --fast --fetch-only
```

## Local Development

```bash
# Dev server (syncs data + serves on :8765)
python dev-serve.py
# → http://localhost:8765

# Dev server with admin panel (no sync, port 8766)
python dev-serve.py --port 8766 --no-sync
# → http://localhost:8766
# → http://localhost:8766/admin/ (Admin Panel)
```

---

## Deploying

```bash
# Deploy frontend + data to Cloudflare Pages
npx wrangler pages deploy app --project-name=peace-paths --skip-caching

# Or let the AI pipeline deploy automatically (runs wrangler after analysis)
python ai-analyze-prod.py --daily
```

> All frontend changes go through Git → GitHub → Cloudflare GitHub integration (auto-deploy).
> Only data files (`solutions.json`, `categories.json`) are deployed via `wrangler`.

## Required Env Vars (`.env`)

| Variable | Description |
|----------|-------------|
| `LLAMA_CPP_URL` | llama.cpp server URL (e.g., `http://192.168.2.121:8080`) |
| `AI_MODEL` | Model name (e.g., `Qwen3.6-27B`) |
| `CLOUDFLARE_API_TOKEN` | API token with `pages_edit` permission |
| `CLOUDFLARE_ACCOUNT_ID` | Numeric Cloudflare account ID |

Copy `.env.example` to `.env` and fill in values.

## Debug Checklist

1. **No data on page?** Check `app/solutions.json` exists. Run `dev-serve.py` to sync.
2. **AI classification failing?** Verify `LLAMA_CPP_URL` in `.env` points to running llama.cpp instance.
3. **Upload fails?** Verify `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` in `.env`.
4. **Wrong categories?** Edit `categories.json` directly or use the admin panel.
5. **Missing feeds?** Copy `rss-feeds.example.json` → `rss-feeds.json` and add feeds.
